## 1. АРХИТЕКТУРА И СТЕК

### Проверенный стек
| Компонент | Технология | Почему |
|-----------|-----------|--------|
| Фреймворк бота | **aiogram 3.15** | Полностью async, middleware, роутеры |
| LLM | **Claude Sonnet** через OpenRouter | Следует инструкциям (GPT-4o — нет) |
| БД | **PostgreSQL + asyncpg** | Масштабируемость, JSONB, полнотекстовый поиск |
| Конфиг | **pydantic-settings** | Валидация, типизация, .env автоматически |
| Документы | **python-docx, pymupdf, openpyxl** | Без внешних зависимостей |
| Поиск | **SerpApi + DuckDuckGo** | Резервирование (DDG бесплатный) |

### GPT-4o vs Claude Sonnet
- **GPT-4o отвергнут**: игнорировал инструкции, выдавал Markdown/LaTeX вместо HTML
- **Claude Sonnet**: точно следует формату, не "фантазирует" инструменты

### Поток обработки сообщения
```
Telegram → aiogram Dispatcher
  → WhitelistMiddleware (outer) — блокировка чужих
  → RateLimitMiddleware (inner) — 10 msg / 5 min
  → InjectionFilterMiddleware (inner) — regex-фильтр инъекций
  → Handler (chat / documents / commands)
  → process_message() — история из PostgreSQL + системный промпт
  → LLMService.chat() — отправка в OpenRouter с tool schemas
  → Tool Calling Loop (до 10 итераций)
  → ResponseValidator — детерминистические проверки
  → split_message() — разбивка если > 4096 символов
  → Telegram — отправка ответа (HTML parse_mode)
```

## 5. TOOL CALLING LOOP (services/llm.py)

### Ключевой паттерн: LLM сам выбирает инструменты
```python
class LLMService:
    def __init__(self):
        self.client = AsyncOpenAI(
            api_key=settings.openrouter_api_key,
            base_url=settings.openrouter_base_url
        )

    async def chat(self, messages, tools=None, tool_executor=None) -> dict:
        for iteration in range(settings.max_tool_iterations):
            try:
                response = await self.client.chat.completions.create(
                    model=settings.llm_model,
                    messages=messages,
                    tools=tools,
                    temperature=settings.temperature,
                    max_tokens=settings.max_tokens
                )
            except Exception as e:
                logger.error(f"LLM error: {e}", exc_info=True)
                return {"content": "Ошибка при обработке запроса.", "files": []}

            # ⚠️ ОБЯЗАТЕЛЬНО: проверить что choices не пуст
            if not response.choices:
                return {"content": "Не удалось получить ответ.", "files": []}

            choice = response.choices[0]
            message = choice.message

            # Если нет tool_calls — финальный ответ
            if not message.tool_calls:
                return {"content": message.content or "", "files": []}

            # ⚠️ Детектирование дублей (защита от бесконечного цикла)
            current_calls = {(tc.function.name, tc.function.arguments) for tc in message.tool_calls}
            if current_calls == previous_calls:
                break  # Те же вызовы = зацикливание
            previous_calls = current_calls

            # Добавляем ответ ассистента в историю
            messages.append({"role": "assistant", "tool_calls": [...]})

            # Выполняем инструменты
            for tool_call in message.tool_calls:
                result = await tool_executor(tool_call.function.name, json.loads(tool_call.function.arguments))

                # ⚠️ Оборачиваем результат в контекст для LLM
                wrapped = f'[Результат "{tool_call.function.name}"]: {json.dumps(result, ensure_ascii=False)}'

                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": wrapped
                })

        return {"content": message.content or "", "files": []}
```

### Критические правила:
1. **Проверяй `response.choices`** — может быть пуст (OpenRouter)
2. **Детектируй дубли** — одинаковые tool_calls 2 раза подряд = зацикливание → break
3. **Оборачивай результат** — LLM нужен контекст откуда данные, не голый JSON
4. **Max iterations** — обязательно ограничивать (10 — оптимально)
5. **None content** — заменяй на пустую строку `""` (некоторые API крашатся)

## 6. MIDDLEWARE — БЕЗОПАСНОСТЬ

### Whitelist (outer middleware)
```python
class WhitelistMiddleware(BaseMiddleware):
    def __init__(self):
        self.allowed = set(settings.allowed_ids_list)

    async def __call__(self, handler, event, data):
        if not self.allowed:  # пустой = все допущены
            return await handler(event, data)

        # ⚠️ from_user может быть None!
        if not event.from_user:
            return

        if event.from_user.id not in self.allowed:
            return  # тихо игнорируем

        return await handler(event, data)
```

### Rate Limit (sliding window)
```python
class RateLimitMiddleware(BaseMiddleware):
    def __init__(self):
        self.requests: dict[int, list[float]] = defaultdict(list)

    async def __call__(self, handler, event, data):
        user_id = event.from_user.id
        now = time.time()
        window = settings.rate_limit_window

        # Чистим старые записи
        self.requests[user_id] = [t for t in self.requests[user_id] if now - t < window]

        if len(self.requests[user_id]) >= settings.rate_limit_messages:
            oldest = self.requests[user_id][0]
            wait = max(1, int(window - (now - oldest)))  # ⚠️ max(1, ...) — не отрицательное
            await event.answer(f"Подождите {wait} сек.")
            return

        self.requests[user_id].append(now)
        return await handler(event, data)
```

### Prompt Injection Filter
```python
INJECTION_PATTERNS = [
    r'ignore\s+(all\s+)?previous\s+instructions',
    r'ты\s+теперь\s+не\s+',
    r'ADMIN\s*OVERRIDE',
    r'покажи\s+(свой\s+)?системн',
    r'system\s*prompt',
    r'забудь\s+(все\s+)?инструкции',
    r'new\s+instructions?\s*:',
    r'режим\s+разработчика',
    r'developer\s+mode',
    r'ты\s+теперь\s+(DAN|джейлбрейк)',
]

class InjectionFilterMiddleware(BaseMiddleware):
    async def __call__(self, handler, event, data):
        text = (event.text or "").lower()
        for pattern in INJECTION_PATTERNS:
            if re.search(pattern, text, re.IGNORECASE):
                await event.answer("Запрос отклонён.")
                return
        return await handler(event, data)
```

def extract_pdf(path: str) -> str:
    """pymupdf4llm → fallback pymupdf"""
    try:
        import pymupdf4llm
        return pymupdf4llm.to_text(path)
    except Exception:
        pass

    # Fallback
    with pymupdf.open(path) as doc:  # ⚠️ context manager!
        return "\n".join(page.get_text() for page in doc)

def extract_docx(path: str) -> str:
    from docx import Document
    doc = Document(path)
    parts = [p.text for p in doc.paragraphs if p.text.strip()]
    # + таблицы
    for table in doc.tables:
        for row in table.rows:
            parts.append(" | ".join(cell.text for cell in row.cells))
    return "\n".join(parts)

## 8. POSTGRESQL — ПАМЯТЬ БОТА

### Схема
```sql
CREATE TABLE IF NOT EXISTS chat_messages (
    id SERIAL PRIMARY KEY,
    chat_id BIGINT NOT NULL,
    role VARCHAR(20) NOT NULL,      -- "user" или "assistant"
    content TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()  -- ⚠️ TIMESTAMPTZ, не TIMESTAMP!
);
CREATE INDEX idx_chat_messages_chat_id ON chat_messages(chat_id, created_at DESC);
```

### Паттерн: asyncpg напрямую (без ORM)
```python
import asyncpg

class DatabaseService:
    def __init__(self, url: str):
        self.url = url
        self.pool: asyncpg.Pool | None = None

    async def init(self):
        self.pool = await asyncpg.create_pool(self.url, min_size=2, max_size=10)
        async with self.pool.acquire() as conn:
            await conn.execute(CREATE_TABLE_SQL)

    async def save_message(self, chat_id: int, role: str, content: str):
        async with self.pool.acquire() as conn:
            await conn.execute(
                "INSERT INTO chat_messages (chat_id, role, content) VALUES ($1, $2, $3)",
                chat_id, role, content[:10000]  # ⚠️ лимит 10K символов
            )

    async def get_history(self, chat_id: int, limit: int = 20) -> list[dict]:
        async with self.pool.acquire() as conn:
            rows = await conn.fetch(
                "SELECT role, content FROM chat_messages WHERE chat_id = $1 ORDER BY created_at DESC LIMIT $2",
                chat_id, limit
            )
        return [{"role": r["role"], "content": r["content"]} for r in reversed(rows)]

    async def trim_history(self, chat_id: int, max_messages: int = 15):
        """Оставить только последние N сообщений"""
        async with self.pool.acquire() as conn:
            await conn.execute("""
                DELETE FROM chat_messages WHERE chat_id = $1 AND id NOT IN (
                    SELECT id FROM chat_messages WHERE chat_id = $1 ORDER BY created_at DESC LIMIT $2
                )
            """, chat_id, max_messages)

    async def close(self):
        if self.pool:
            await self.pool.close()
```

### Правила:
- **TIMESTAMPTZ** — всегда (не TIMESTAMP, иначе таймзона теряется)
- **Лимит контента** — 10K символов (не хранить весь PDF)
- **Trim** — после каждого сообщения (15 последних), иначе контекст LLM переполнится
- **Пул** — min 2, max 10 (не создавать connection на каждый запрос)

## 9. ВАЛИДАТОР ОТВЕТОВ (вместо второго LLM)

### Почему не AI-Ревизор?
Второй вызов LLM = +3-5 секунд + двойная стоимость. Детерминистические проверки быстрее и дешевле.

### Паттерн: Regex-валидация
```python
class ResponseValidator:
    def validate(self, text: str, user_question: str) -> dict:
        issues = []
        fixed = text

        # КФ-1: Есть ли ссылки на статьи?
        if not re.search(r'ст\.\s*\d+', fixed):
            issues.append("Нет ссылок на статьи закона")

        # КФ-2: Убрать перенаправления к юристу
        redirect_phrases = [
            "обратитесь к юристу",
            "рекомендую обратиться к специалисту",
            "проконсультируйтесь с адвокатом",
        ]
        for phrase in redirect_phrases:  # ⚠️ БЕЗ break — удаляем ВСЕ
            fixed = fixed.replace(phrase, "")

        # КФ-5: Markdown артефакты
        # ⚠️ Одиночный "- " — НЕ Markdown. Нужно 2+ строки подряд
        if re.search(r'(^|\n)-\s+.+\n-\s+', fixed):
            issues.append("Markdown списки")
            fixed = re.sub(r'^-\s+', '• ', fixed, flags=re.MULTILINE)

        # КФ-5b: LaTeX
        if re.search(r'\\(text|frac|sum|int)\{', fixed):
            issues.append("LaTeX формулы")

        # Чистка пробелов после удалений
        fixed = re.sub(r' {2,}', ' ', fixed)  # ⚠️ Двойные пробелы

        return {"is_valid": len(issues) == 0, "issues": issues, "fixed_text": fixed}
```

## 10. РАЗБИВКА ДЛИННЫХ СООБЩЕНИЙ

### Telegram лимит: 4096 символов

```python
def split_message(text: str, max_length: int = 4096) -> list[str]:
    """Умная разбивка с балансировкой HTML-тегов"""

    if not text.strip():
        return []  # ⚠️ Пустой текст → пустой список (не [""])

    if len(text) <= max_length:
        return [text]

    chunks = []
    while text:
        if len(text) <= max_length:
            chunks.append(text)
            break

        # Ищем точку разбивки (приоритет: \n\n > \n > . > пробел)
        cut = max_length
        for separator in ['\n\n', '\n', '. ', ' ']:
            pos = text.rfind(separator, 0, max_length)
            if pos > max_length * 0.3:  # не слишком короткий кусок
                cut = pos + len(separator)
                break

        chunk = text[:cut]
        text = text[cut:]

        # Балансировка HTML тегов
        chunk = balance_html_tags(chunk)
        chunks.append(chunk)

    return chunks
```

## 11. КЭШИРОВАНИЕ

### Паттерн: In-memory LRU с ограничением
```python
_cache: dict[str, dict] = {}
MAX_CACHE = 500
CACHE_TTL = 86400  # 24 часа

def _get_from_cache(key: str) -> dict | None:
    if key in _cache:
        entry = _cache[key]
        if time.time() - entry["ts"] < CACHE_TTL:
            return entry["data"]
        del _cache[key]
    return None

def _put_to_cache(key: str, data: dict):
    if not data:  # ⚠️ НЕ кэшировать пустые результаты!
        return
    if len(_cache) >= MAX_CACHE:  # ⚠️ ОБЯЗАТЕЛЬНО лимит!
        oldest = min(_cache, key=lambda k: _cache[k]["ts"])
        del _cache[oldest]
    _cache[key] = {"data": data, "ts": time.time()}
```

### Правила:
1. **Лимит размера** — без него кэш растёт бесконечно
2. **Не кэшировать пустое** — пустой результат может быть временной ошибкой
3. **TTL** — данные устаревают (24ч для поиска — ок)

## 13. ПАРСИНГ ЧИСЕЛ ОТ LLM

### Проблема: LLM может вернуть числа в разных форматах

```python
# ❌ Крашится
float("100 000")    # ValueError
float("1,5")        # ValueError
float("₽ 50000")    # ValueError

# ✅ Универсальный парсер
def _parse_number(value) -> float:
    s = str(value).strip()
    s = re.sub(r'[^\d.,\-]', '', s)  # убрать всё кроме цифр, точки, запятой
    s = re.sub(r'\s+', '', s)         # убрать пробелы
    s = s.replace(',', '.')            # запятая → точка
    return float(s)
```

## 15. АТОМАРНАЯ ЗАПИСЬ ФАЙЛОВ

### Проблема: Краш во время записи = битый файл
```python
import tempfile
import os
import threading

_lock = threading.Lock()

def save_json(filepath: str, data: dict):
    with _lock:  # ⚠️ Защита от race condition
        # Пишем во временный файл
        dir_path = os.path.dirname(filepath)
        with tempfile.NamedTemporaryFile('w', dir=dir_path, suffix='.tmp', delete=False) as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
            tmp_path = f.name

        # Атомарная замена
        os.replace(tmp_path, filepath)  # ⚠️ replace, не rename
```

## 16. СИСТЕМНЫЙ ПРОМПТ — ЛУЧШИЕ ПРАКТИКИ

### Структура (проверено на 6 версиях):
1. **Абсолютные правила** — формат, что НЕЛЬЗЯ делать (первый блок!)

4. **Методология** — как анализировать, по каким шагам
5. **Чеклист** — конкретные пункты проверки
6. **Формулы** — расчёты (если есть)
7. **Структура ответа** — длина, формат, уровни уверенности
8. **Справочные данные** — числа, ставки, тарифы

### Правила:
- **Размер**: 300-500 строк оптимально. >1000 — LLM теряет фокус
- **Переменные из config**: `prompt.format(key_rate=settings.key_rate_percent)`
- **⚠️ Экранирование**: Если в промпте есть `{}` (формулы, LaTeX) — экранировать как `{{}}`
  ```python
  # ❌ Крашит .format()
  "Формула: \frac{a}{b}"

  # ✅ Экранировано
  "Формула: \\frac{{a}}{{b}}"
  ```

## 19. ВСЕ БАГИ И УРОКИ (34+)

### LLM / API
| Баг | Решение | Правило |
|-----|---------|---------|
| Пустой `response.choices` | `if not response.choices:` guard | Всегда проверять ответ API |
| Дубли tool_calls = бесконечный цикл | Детектировать (name, args) дубли | Break при повторе |
| Tool result без контекста | Оборачивать в `[Результат "name"]` | LLM нужен контекст |
| `None` content от API | Заменять на `""` | Проверять тип |

### Безопасность
| Баг | Решение | Правило |
|-----|---------|---------|
| `str(e)` утечка API-ключа | Generic сообщение + logger | НИКОГДА `str(e)` пользователю |
| `from_user` может быть None | `if not event.from_user:` guard | Проверять все поля |
| Пустой whitelist = все | Документировать поведение | Явно проверять |

### Async / Ресурсы
| Баг | Решение | Правило |
|-----|---------|---------|
| Sync I/O блокирует event loop | `asyncio.to_thread()` | Всегда для sync в async |
| Ресурс не закрывается | `with` / `try-finally` | Context manager всегда |
| `create_task` теряет ошибки | Safe wrapper | Обернуть в try-except |
| import внутри цикла | Перенести в начало файла | Импорты = начало файла |

### БД
| Баг | Решение | Правило |
|-----|---------|---------|
| TIMESTAMP без таймзоны | TIMESTAMPTZ | Всегда TIMESTAMPTZ |

### Парсинг
| Баг | Решение | Правило |
|-----|---------|---------|
| `float("100 000")` крашится | `_parse_number()` | Всегда парсить числа от LLM |
| Русское склонение суффиксом | Полная форма | "Генерального директора", не "директора" |

### Кэш
| Баг | Решение | Правило |
|-----|---------|---------|
| Кэш растёт бесконечно | Лимит 500 записей | Любой кэш = лимит |
| Кэш пустых результатов | `if results:` перед кэшем | Не кэшировать ошибки |

### Валидатор
| Баг | Решение | Правило |
|-----|---------|---------|
| `break` после первой фразы | Убрать break, удалять ВСЕ | Цикл удаления — без break |
| Одиночный `- ` = Markdown | 2+ строки подряд | Ужесточить regex |
| Двойные пробелы после удаления | `re.sub(r' {2,}', ' ')` | Чистить после замен |

### Файлы / JSON
| Баг | Решение | Правило |
|-----|---------|---------|
| Race condition при записи | `threading.Lock` + atomic write | Lock для read-modify-write |
| ID дубли после FIFO-обрезки | `max(existing) + 1` | Монотонные ID |
| Коррекция ложноположительная | Проверять только первые 50 символов | Ограничивать паттерн |

### Поиск
| Баг | Решение | Правило |
|-----|---------|---------|
| HTTP статус не проверяется | `if resp.status != 200:` ДО обработки | Всегда проверять статус |
| duckduckgo-search v7+ сломан | Убрать `with DDGS()`, backends→auto | Следить за changelog |

### Rate Limit
| Баг | Решение | Правило |
|-----|---------|---------|
| Отрицательные секунды ожидания | `max(1, int(...))` | Clamp к минимуму |

## 21. ПАТТЕРНЫ ДЛЯ ПОВТОРНОГО ИСПОЛЬЗОВАНИЯ

1. **Pydantic BaseSettings** — конфиг из .env с валидацией
2. **Middleware стек** — whitelist → rate limit → injection filter
3. **Tool calling loop** — LLM сам выбирает инструменты (до N итераций)
4. **ResponseValidator** — regex-проверки вместо второго LLM
5. **asyncio.to_thread()** — sync I/O в async контексте

7. **Smart message splitting** — приоритетные разделители + балансировка HTML
8. **LRU cache** — размер, TTL, не кэшировать пустое
9. **Safe background tasks** — try-except wrapper для create_task
10. **Centralized config** — все переменные в одном месте
11. **Generic error messages** — logger.error для себя, "попробуйте позже" для пользователя
12. **Number parsing** — убрать пробелы, запятую → точку
13. **Sliding window rate limit** — defaultdict(list) + time.time()
14. **Dual search** — SerpApi (точный) + DuckDuckGo (бесплатный fallback)
15. **Client profiles** — фоновая суммаризация каждые N сообщений
