# Скрининг акций по критериям

## Когда использовать
Пользователь просит найти акции по критериям: "недооцененные", "дивидендные", "с низким долгом", "ростовые".

## Алгоритм

### 1. Определи критерии
Типичные фильтры:

**Стоимостные (Value):**
- P/E < 7
- P/BV < 1
- EV/EBITDA < 5

**Дивидендные (Income):**
- Дивидендная доходность > 10%
- Стабильная история выплат

**Низкий долг:**
- ND/EBITDA < 1 (или отрицательный = нет долга)

**Эффективные:**
- ROE > 20%
- Маржинальность EBITDA > 30%

**Ростовые (Growth):**
- Рост выручки > 15% г/г
- P/S < 2

### 2. Фильтрация
Загрузи @knowledge/companies_data.json и отфильтруй компании по заданным критериям.

**Обработка отсутствующих полей:**
- Если поле = null для компании — пропустить её (не включать и не исключать)
- Для критериев Magic Formula (EBIT, Invested Capital): использовать поле `ebit` из JSON; если нет — приблизить через EBITDA × 0.85
- Для Payout Ratio: использовать поле `payout_ratio` из JSON
- Для Debt/Equity: использовать поле `debt_equity` из JSON
- Для Graham Number: использовать `eps` и `bvps` из JSON
- Банки автоматически исключаются из Magic Formula и Z-Score (по Гринблатту и Альтману)
- **Аномалии данных:** если P/E < 0.5 или P/BV < 0.05 — пометить как "аномальные данные, проверить" (пример: Россети с P/E=0.11 из-за огромного кол-ва акций)
- Для Z-Score: использовать `total_assets`, `working_capital`, `ebit` из JSON. Формула: Z = 1.2×(WC/Assets) + 1.4×(NI/Assets) + 3.3×(EBIT/Assets) + 0.6×(Mcap/Debt) + 1.0×(Revenue/Assets)
- Для F-Score: использовать `ocf` из JSON для проверки OCF > NI. Для delta-критериев — если нет прошлогодних данных, считать только доступные (4 из 9)
