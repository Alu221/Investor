# Сигналы покупки и продажи

## Когда использовать
Пользователь спрашивает "когда покупать?", "пора продавать?", "есть ли сигналы?"

## Алгоритм

### 1. Сигналы ПОКУПКИ (фундаментальные)
- P/E упал ниже 3-летнего среднего компании
- ND/EBITDA снизился (долг уменьшается)
- Объявлены дивиденды выше ожиданий
- F-Score вырос до 7+ (финансовое здоровье улучшается)
- Компания попала в Magic Formula топ (ROIC + EY)
- Цена < Graham Number (глубокая недооценка)
- Сектор на дне цикла, но фундамент крепкий (Marks/Templeton)

### 2. Сигналы ПРОДАЖИ (красные флаги)
- ND/EBITDA > 3x и продолжает расти
- Менеджмент меняется на худший
- Потеря moat (конкуренты отбирают долю)
- M-Score > -2.22 (подозрение на манипуляции)
- Z-Score < 1.8 (риск банкротства)
- P/E > 25 без обоснованного роста
- Дивиденды срезали без обоснования

### 3. Макро-сигналы
- ЦБ начинает снижать ставку → покупать акции и длинные ОФЗ
- ЦБ резко повышает ставку → короткие облигации и кэш
- Нефть < $50 → осторожность с нефтегазом
- Рубль резко слабеет → покупать экспортёров

### 4. Циклические сигналы (Marks)
- Все говорят "покупай!" + VIX < 12 → время осторожности
- Паника + VIX > 30 → время смотреть покупки
- Second-level thinking: если все продают — может, пора покупать?

### 5. Количественные проверки из JSON

**Graham Number** (сигнал покупки если цена < GN):
```
GN = √(22.5 × eps × bvps)   ← поля eps, bvps из companies_data.json
Если price < GN → ПОКУПКА по Грэму
```

**Z-Score Альтмана** (сигнал продажи если Z < 1.8):
```
Z = 1.2×(working_capital/total_assets) + 1.4×(net_income/total_assets)
    + 3.3×(ebit/total_assets) + 0.6×(mcap/(total_assets-equity))
    + 1.0×(revenue/total_assets)
Z > 3.0 → безопасно | 1.8-3.0 → серая зона | < 1.8 → ПРОДАЖА (банкротство)
```

**PEG Lynch** (сигнал если PEG < 1):
```
PEG = (p_e) / (eps_growth_yoy × 100)   ← eps_growth_yoy из JSON
PEG < 1.0 → ПОКУПКА по Линчу
```

**Таргет InveStory** (если есть target_price в JSON):
```
target_upside > 20% → потенциальная ПОКУПКА
target_upside < -10% → потенциальная ПРОДАЖА (переоценена)
```

### 6. Для конкретной компании
Проанализируй текущее состояние и определи:
- Рассчитай Graham Number, Z-Score, PEG (если данные есть)
- Проверь target_price и target_upside
- Есть ли сейчас сигнал покупки? (какой именно)
- Есть ли сигнал продажи? (какой именно)
- Или нейтрально (держать/наблюдать)
