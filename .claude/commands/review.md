# /review — Ревью кода

Проведи ревью кода. Если есть git изменения — проверяй их. Если нет — проверяй весь проект.

## Шаг 1: Определи scope

```bash
git diff --staged 2>/dev/null
git diff 2>/dev/null
git status 2>/dev/null
```

- Есть изменения → проверяй только изменённые файлы
- Нет изменений → проверяй все файлы с кодом в проекте (src/, *.py, *.js, *.ts)

## Шаг 2: Проверь по критериям

### 1. Читаемость
- Понятные названия переменных и функций?
- Есть ли комментарии где логика неочевидна?
- Не слишком ли длинные функции (>50 строк)?
- Нет ли дублирования кода?
- Нет ли мёртвого кода (закомментированные блоки, неиспользуемые импорты)?

### 2. Ошибки
- Обрабатываются ли ошибки (try/except, if/else)?
- Нет ли очевидных багов?
- Нет ли print()/console.log() для дебага?
- Существующие тесты не сломаны?

### 3. Безопасность
- Нет ли паролей/токенов/API-ключей в коде?
- Нет ли .env в staging?
- Проверяются ли входные данные (user input, API responses)?
- Нет ли SQL-инъекций, XSS, command injection?

### 4. Соответствие задаче (только если есть git diff)
- Изменения относятся к текущей задаче?
- Нет "заодно поправил"?
- Каждый коммит = одна цель?

## Шаг 3: Линтер и тесты

Определи стек из CLAUDE.md и запусти:

**Python:**
```bash
ruff check . 2>/dev/null || python -m py_compile src/*.py 2>/dev/null
pytest tests/ -q 2>/dev/null || echo "Тесты не настроены"
```

**Node.js:**
```bash
npx eslint . 2>/dev/null || echo "ESLint не настроен"
npm test 2>/dev/null || echo "Тесты не настроены"
```

## Шаг 4: Вердикт

Формат ответа — СТРОГО по этому шаблону:

```
## Результат ревью

### Читаемость
✅ или ⚠️ или ❌ — [конкретное описание]

### Ошибки
✅ или ⚠️ или ❌ — [конкретное описание]

### Безопасность
✅ или ⚠️ или ❌ — [конкретное описание]

### Соответствие задаче
✅ или ⚠️ или ❌ — [конкретное описание]

### Общая оценка: OK / ЗАМЕЧАНИЯ / БЛОКЕР

**Главное что исправить:**
1. {файл:строка} — {проблема} — {как исправить}

**Рекомендуемое сообщение коммита** (если есть изменения):
{тип}: {описание}
```

## Правила

- ✅ — всё хорошо
- ⚠️ — замечание, можно коммитить но лучше исправить
- ❌ — БЛОКЕР, нельзя коммитить пока не исправлено
- Будь конкретен: "строка 42 в parser.py" а не "где-то есть проблема"
- Если найден БЛОКЕР — НЕ рекомендуй коммитить
- Найденные проблемы записать в tasks/lessons.md
