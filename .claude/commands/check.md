# /check — "Ты уверен?" Перепроверка перед финализацией

Пользователь считает что задача готова. Твоя роль — адвокат дьявола. Проверь ВСЁ прежде чем подтвердить.

## Контекст

Прочитай `tasks/todo.md` чтобы понять какая задача считается выполненной.

## Чеклист перепроверки

### 1. Критерии готовности выполнены?
- Найди "Готово =" в `tasks/todo.md` — это главные критерии
- Перечитай исходное ТЗ / описание задачи
- Сравни критерии с тем что реально сделано
- Вопрос: "Все ли пункты из 'Готово =' закрыты? Не забыл ли чего?"
- Если критериев нет — спроси пользователя: "Когда задача готова?"

### 2. Код работает?
Запусти проект и проверь основной сценарий:
```bash
# Определи команду запуска из CLAUDE.md
# Запусти и проверь что нет ошибок при старте
```

Если это бот — проверь что стартует без ошибок.
Если парсер — проверь на одном URL.
Если скрипт — запусти с тестовыми данными.

### 3. Тесты проходят?
```bash
# Python
pytest tests/ -v 2>/dev/null || echo "Тесты не настроены"

# Node.js
npm test 2>/dev/null || echo "Тесты не настроены"
```

### 4. Нет мусора?
```bash
git status
git diff --stat
```
- Нет лишних файлов в staging?
- Нет забытых TODO/FIXME/HACK в коде?
- Нет debug-принтов?

### 5. Документация актуальна?
- `CLAUDE.md` — отражает текущий стек и команды?
- `docs/architecture.md` — соответствует реальности?
- `tasks/todo.md` — задача отмечена как выполненная?

### 6. Длинная задача (progress.md)
Если существует `tasks/progress.md`:
- Все шаги должны быть ✅
- Если есть незавершённые шаги — задача НЕ готова
- Если все шаги ✅ — напомни удалить progress.md

### 7. Edge cases
Подумай о граничных случаях для конкретной задачи:
- Что если входные данные пустые?
- Что если API не отвечает?
- Что если файл не найден?
- Что если пользователь ввёл ерунду?

## Формат ответа

```
## Проверка: "Ты уверен?"

### Пройдено
- [x] {что проверено и OK}

### Проблемы
- [ ] {что не так — конкретно}

### Вердикт
{ГОТОВО / НЕ ГОТОВО — {что доделать}}
```

## Правила

- Будь придирчивым. Лучше найти проблему сейчас чем после сдачи клиенту
- Не принимай "потом доделаю" — если важно, доделай СЕЙЧАС
- Если всё OK — так и скажи: "Проверил, всё чисто, можно коммитить/сдавать"
- Проверяй РЕАЛЬНО — запускай код, а не просто читай его глазами
