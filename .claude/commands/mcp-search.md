# /mcp-search — Поиск MCP-серверов

Найди MCP-сервер для указанной задачи/сервиса. Аргумент: $ARGUMENTS

## Порядок поиска (строго по приоритету):

### 1. Официальный реестр Anthropic (ПЕРВЫЙ)
```
URL: https://registry.modelcontextprotocol.io
```
- Проверенные серверы от реальных компаний (Notion, Slack, Figma, etc.)
- Если нашёл тут — бери и не ищи дальше
- WebFetch или WebSearch: `site:registry.modelcontextprotocol.io {запрос}`

### 2. awesome-mcp-servers (GitHub)
```
URL: https://github.com/punkpeye/awesome-mcp-servers
```
- Курируемый список от сообщества
- WebFetch README.md и ищи по категориям
- WebSearch: `site:github.com awesome-mcp-servers {запрос}`

### 3. Smithery
```
URL: https://smithery.ai
```
- Тысячи серверов, удобная установка
- WebSearch: `site:smithery.ai {запрос}`
- Для экспериментов ОК, для продакшена проверяй код

### 4. mcp.so
```
URL: https://mcp.so
```
- Самый большой каталог
- Смотри раздел "Featured" — единственное что проверено
- WebSearch: `site:mcp.so {запрос}`

### 5. PulseMCP
```
URL: https://pulsemcp.com
```
- Каталог + аналитика + тренды
- WebSearch: `site:pulsemcp.com {запрос}`

## Алгоритм

1. Пойми что ищет пользователь (сервис, задача, интеграция)
2. Ищи по каталогам сверху вниз — нашёл в официальном → стоп
3. Для каждого найденного сервера покажи:
   - Название и ссылка
   - Откуда (какой каталог)
   - Что делает (1 строка)
   - Как установить (npx/pip команда если есть)
   - Уровень доверия: ОФИЦИАЛЬНЫЙ / КУРИРУЕМЫЙ / СООБЩЕСТВО

## Формат ответа

```
## MCP-серверы для: {запрос}

### {Название сервера}
- Источник: {каталог} ({уровень доверия})
- Описание: {что делает}
- Установка: `{команда}`
- Ссылка: {url}

### Рекомендация
{какой выбрать и почему}
```

## Правила

- Официальный реестр > всё остальное
- Не рекомендуй серверы без исходного кода (closed source)
- Предупреждай если сервер требует API-ключи
- Если ничего не нашёл — скажи прямо, не выдумывай
