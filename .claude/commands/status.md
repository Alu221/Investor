# /status — Статус проекта

Покажи пользователю полную картину состояния проекта. Быстро, по делу, без воды.

## Сбор данных

Прочитай и проанализируй:

1. **Git-статус:**
```bash
git branch --show-current 2>/dev/null || echo "Git не инициализирован"
git log --oneline -5 2>/dev/null
git status --short 2>/dev/null
git diff --stat 2>/dev/null
```

2. **Файлы проекта:**
- `tasks/todo.md` — задачи
- `tasks/handoff.md` — последний контекст
- `tasks/lessons.md` — уроки
- `CLAUDE.md` — описание проекта
- `knowledge/_index.md` — каталог базы знаний (если есть)
- `tasks/progress.md` — прогресс длинной задачи (если есть)

## Формат вывода

```
## Статус: {название проекта}

**Ветка**: {ветка} | **Последний коммит**: {хеш} {сообщение}

### Задачи
В работе: {N}
- [ ] {задача 1}
- [ ] {задача 2}

Сделано: {N}
- [x] {задача 1}
- [x] {задача 2}

Бэклог: {N}
- [ ] {задача 1}

### Незакоммиченные изменения
{список файлов или "Всё чисто"}

### Последние коммиты
{5 последних коммитов одной строкой}

### Уроки за проект
{количество записей в lessons.md}
{последний урок если есть}

### База знаний
{N статей в knowledge/ или "Пусто"}

### Длинная задача
{Если tasks/progress.md существует: текущий шаг и прогресс. Если нет: "Нет активной"}

### Следующий шаг
{первая незавершённая задача из todo.md}
```

## Правила

- Если файла нет — не падай, просто укажи "файл отсутствует"
- Если git не инициализирован — скажи об этом
- Максимум конкретики, минимум слов
- Это команда для быстрого взгляда — не нужно расписывать подробности
