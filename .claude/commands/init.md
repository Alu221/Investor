# /init — Инициализация нового проекта

Ты запускаешь инициализацию нового проекта. Выполни все шаги последовательно.

## Шаг 1: Bootstrap-интервью

Спроси у пользователя (если не указано). Можно одним блоком:
1. **Название проекта** — короткое, латиницей, через дефис (пример: `kwork-parser`)
2. **Тип** — один из: `bot`, `parser`, `site`, `script`, `api-service`
3. **Стек** — язык и основные библиотеки (пример: `Python 3.12 + aiogram 3 + SQLite`)
4. **Что делает** — одно предложение
5. **Цель** — зачем этот проект, какую проблему решает
6. **Готово =** — когда проект считается завершённым (2-3 критерия)

Если пользователь уже дал эту информацию — НЕ переспрашивай, используй её.
Критерии "Готово =" записать в `tasks/todo.md` в секцию "Критерии готовности".

## Шаг 2: Создание структуры

Создай папки и файлы:

```
{название}/
├── src/                  # Основной код
├── tests/                # Тесты
├── docs/
│   └── architecture.md   # Из шаблона TEMPLATE_docs_architecture.md
├── tasks/
│   ├── todo.md           # Из шаблона TEMPLATE_tasks_todo.md
│   ├── lessons.md        # Из шаблона TEMPLATE_tasks_lessons.md
│   └── handoff.md        # Из шаблона TEMPLATE_tasks_handoff.md
├── knowledge/
│   └── _index.md         # Из шаблона TEMPLATE_knowledge_index.md
├── .env.example          # Из шаблона TEMPLATE_env_example
├── .gitignore            # Из шаблона TEMPLATE_gitignore
└── CLAUDE.md             # Из шаблона TEMPLATE_CLAUDE.md
```

Во всех файлах замени `{PROJECT_NAME}` на реальное название проекта.
В CLAUDE.md также заполни тип, стек и описание.

## Шаг 3: Git

```bash
git init
git add -A
git commit -m "chore: init project {название}"
```

## Шаг 4: Первая задача

Спроси пользователя: "Какая первая задача?" и добавь её в `tasks/todo.md`.

## Шаг 5: Отчёт

Покажи что создано:
- Список файлов
- Напомни: "Заполни .env по образцу .env.example"
- Напомни: "Дополни docs/architecture.md когда определишься с архитектурой"

## Важно

- Используй шаблоны из папки `учение/` (TEMPLATE_*) если они доступны
- Если шаблонов нет — создавай файлы с разумным содержимым по аналогии
- НЕ создавай лишних файлов — только то что перечислено выше
- НЕ ставь зависимости — это отдельный шаг
- `tasks/progress.md` НЕ создаётся при init — только когда появится задача 5+ шагов (шаблон: TEMPLATE_tasks_progress.md)
