# /handoff — Передача контекста при завершении сессии

Создай или обнови файл `tasks/handoff.md` чтобы следующая сессия (или другой агент) мог продолжить работу без потери контекста.

## Шаг 1: Собери данные

Выполни и проанализируй:

```bash
# Текущая ветка и статус
git branch --show-current
git status --short

# Последние коммиты этой сессии
git log --oneline -10

# Незакоммиченные изменения
git diff --stat
```

Прочитай:
- `tasks/todo.md` — что в работе, что сделано
- `tasks/lessons.md` — были ли новые уроки в этой сессии

## Шаг 2: Обнови handoff.md

Заполни файл `tasks/handoff.md` конкретными данными:

```markdown
# Передача контекста: {название проекта}

## Последнее обновление: {сегодняшняя дата}

## Что сделано в этой сессии
- {конкретный список с результатами, не просто "работал над X"}

## Что дальше
1. {следующий шаг — самый приоритетный}
2. {второй шаг}
3. {третий шаг}

## Текущее состояние
- **Ветка**: {реальная ветка из git}
- **Работает ли**: {да/нет, что конкретно сломано}
- **Незакоммиченные изменения**: {есть/нет, какие}
- **Блокеры**: {что мешает, или "нет"}

## Договорённости с пользователем
- {решения принятые в этой сессии}

## Контекст который легко потерять
- {неочевидные вещи: workaround-ы, особенности API, нюансы}
- {почему что-то сделано необычным способом}

## Memory flush
> Важное из контекста сессии, что потеряется при обрезке/рестарте.
- {ключевые решения и их причины}
- {факты о проекте, которые ещё не в CLAUDE.md}
- {промежуточные результаты незавершённых задач}
- {что пробовали и почему не сработало}
```

## Шаг 3: Финальные действия

1. Проверь что нет незакоммиченных важных изменений
2. Если есть — предложи закоммитить
3. Обнови `tasks/todo.md` — актуальные статусы задач

## Шаг 4: Покажи пользователю

Выведи краткую сводку:
```
Сессия завершена.
Сделано: {1-2 главных результата}
Следующий шаг: {что делать в следующей сессии}
Незакоммиченное: {есть/нет}
```

## Правила

- Пиши КОНКРЕТНО: "добавил парсер цен с сайта X" а не "работал над парсером"
- Не теряй контекст — если было сложное решение, запиши ПОЧЕМУ так
- Если сессия прервалась на середине — укажи на каком ТОЧНО шаге остановились
- Handoff должен быть достаточным чтобы ДРУГОЙ агент мог продолжить без вопросов
