# Архитектура: ИНвестбот

## Общая схема

```
Telegram → iia (Node.js) → Claude Code (Opus) → Ответ
                              ↓
                    CLAUDE.md + knowledge/ + skills/
                              ↓
                    Инвестиционный анализ
```

## Компоненты

### 1. iia (платформа)
- **Назначение**: Связывает Telegram с Claude Code
- **Файлы**: `~/.iia/bot/index.js` (закрытый)
- **Зависимости**: Node.js, Telegram Bot API, Claude API, Deepgram

### 2. CLAUDE.md (мозг бота)
- **Назначение**: Определяет роль, знания и правила ответов
- **Файлы**: `CLAUDE.md`
- **Зависимости**: knowledge/, skills/

### 3. Knowledge Base (база знаний)
- **Назначение**: Структурированные знания по фундаментальному анализу
- **Файлы**: `knowledge/*.md`, `knowledge/companies_data.json`
- **Зависимости**: Учебные материалы (Инвестиции/)

### 4. Skills (скиллы)
- **Назначение**: Специализированные команды для анализа
- **Файлы**: `.claude/skills/*.md`
- **Зависимости**: knowledge/

## Внешние сервисы и API

| Сервис | Назначение | Документация |
|--------|-----------|--------------|
| iia | Telegram ↔ Claude Code | Настройка через /settings |
| Claude API | AI-ответы (Opus) | Через подписку iia |
| Deepgram | STT (голосовые) | Встроено в iia |
| MOEX | Данные по акциям | moex.com (ручное обновление пока) |

## Решения и причины

| Решение | Почему | Альтернативы |
|---------|--------|--------------|
| iia вместо aiogram | Готовая платформа, быстрый старт | aiogram (больше кода), n8n (ограничения) |
| Claude Opus | Лучшее следование инструкциям | Sonnet (дешевле, менее точный) |
| Markdown knowledge | Claude читает нативно | PostgreSQL RAG (избыточно на старте) |
| JSON для данных компаний | Простой, читаемый, легко обновлять | SQLite (избыточно) |

## Ограничения и риски

- Данные компаний обновляются вручную (нет автопарсинга MOEX пока)
- iia закрытая платформа — нет контроля над index.js
- Лимиты API Claude через iia подписку
- Telegram лимит: 4096 символов на сообщение
